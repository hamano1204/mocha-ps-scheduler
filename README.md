# Mocha PS Scheduler

Windows のシステムトレイ（タスクトレイ）に常駐し、設定されたスケジュール（Cron 式）に従って、PowerShell スクリプトなどのジョブをバックグラウンドで自動実行する、軽量かつ頑強なスケジューラアプリケーションです。

---

## 🌟 主な機能

1. **コンソールの完全非表示実行**
   - ジョブ実行用のコンソールウィンドウを一瞬たりとも表示させることなく、完全にバックグラウンドでプロセスを起動します。
2. **秒単位のスケジュール指定 (6フィールド Cron)**
   - 秒・分・時・日・月・曜日を指定可能な6フィールド形式の Cron 記法（例: `*/5 * * * * *` で5秒ごと）をサポートしています。
3. **ホットリロード (設定ファイルのリアルタイム監視)**
   - アプリケーションが起動した状態で `config.json` を変更・保存すると、自動的に再読み込みしてスケジュールが即時反映されます。アプリの再起動は不要です。
4. **二重起動防止 & 排他制御**
   - 同一ジョブの多重実行を自動的に防ぎます。既に実行中のジョブはトリガーされてもスキップされます。
5. **タイムアウト & 自動リトライ**
   - ジョブごとに実行タイムアウト時間を指定可能。タイムアウト時や手動キャンセル時には無駄なハングを防ぐため、即座にリトライを打ち切ります。通常のエラー終了時は、設定された回数とディレイ（遅延秒数）に基づいて再試行します。
6. **ログ管理の独立化**
   - 実行したジョブごとの標準出力（Stdout）および標準エラー（Stderr）は、リアルタイムに専用ログファイル (`logs/job-{jobId}.log`) に書き出されます。
7. **トースト通知**
   - ジョブの成功、または失敗時に Windows ネイティブのトースト通知でステータスをお知らせします（設定により有効/無効を切り替え可能）。
8. **スタートアップ自動起動**
   - システムトレイのメニューから、Windows 起動時に自動実行する設定（レジストリ登録）をワンクリックでトグルできます。

## 🚀 クイックスタート

### 動作環境
- .NET 10 以上
- Windows OS (Windows 10 / 11 推奨)

### 1. ビルドと実行
リポジトリのルートディレクトリで以下のコマンドを実行します。

```powershell
# 依存関係の復元とビルド
dotnet build

# アプリケーションの実行
dotnet run
```

起動すると、Windows のシステムトレイにアイコンが常駐します。

### 2. 設定ファイルの構成 (`config.json`)
アプリケーションの実行ディレクトリ内に `config.json` が配置されます（初回起動時に自動生成されます）。

#### 設定例
```json
{
  "startup": true,
  "notification": {
    "onSuccess": false,
    "onFailure": true
  },
  "jobs": [
    {
      "id": "sample-ps",
      "name": "サンプルPowerShellジョブ",
      "type": "PowerShell",
      "scriptPath": "C:\\path\\to\\your\\script.ps1",
      "arguments": "-Argument1 'Value'",
      "schedule": "*/30 * * * * *",
      "timeoutSeconds": 60,
      "retryCount": 1,
      "retryDelaySeconds": 10,
      "enabled": true
    }
  ]
}
```

#### 各設定項目の説明
| 項目 | 型 | 説明 |
| :--- | :--- | :--- |
| `startup` | bool | スタートアップへの登録有無。 |
| `notification` | object | 通知設定（`onSuccess` = 成功時通知, `onFailure` = 失敗時通知）。 |
| `jobs` | array | 実行するジョブのリスト。 |
| `jobs[].id` | string | ジョブのユニークID。ログファイル名等にも使用されます。 |
| `jobs[].name` | string | 画面に表示されるジョブの名称。 |
| `jobs[].type` | string | ジョブの種類（`PowerShell` のみサポート。将来拡張可能）。 |
| `jobs[].scriptPath` | string | 実行するスクリプトファイルへのフルパス。 |
| `jobs[].arguments` | string | スクリプトへ渡す引数。 |
| `jobs[].schedule` | string | **6フィールド形式**の Cron 設定 (`秒 分 時 日 月 曜日`)。 |
| `jobs[].timeoutSeconds` | int | ジョブの最大実行時間（秒）。0でタイムアウト無効。 |
| `jobs[].retryCount` | int | ジョブ失敗時のリトライ回数。 |
| `jobs[].retryDelaySeconds`| int | リトライまでの間隔（秒）。 |
| `jobs[].enabled` | bool | ジョブの有効/無効の切り替え。 |

## 📂 フォルダ構成とログについて

- **設定ファイル**: アプリケーション実行バイナリと同階層にある `config.json`
- **ログフォルダ**: アプリケーション実行バイナリと同階層の `logs/` フォルダ以下に保存されます。
  - `logs/app.log`: アプリケーション全体の起動/停止、設定ファイル再読み込みなどのシステムログ。
  - `logs/job-{jobId}.log`: 各ジョブの出力（PowerShell の書き出しやエラー）をリアルタイムに記録する専用ログ。
